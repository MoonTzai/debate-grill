# Debate-Coach 隐私政策

更新日期：2026年7月10日

本隐私政策适用于 `Debate-Coach` iOS 应用（以下简称“本应用”）。

我们重视你的隐私。根据当前版本的产品设计，本应用遵循以下原则：

- 尽量在本地设备完成数据存储与处理
- 由用户自行配置第三方模型接口
- 不通过开发者自建服务器中转聊天内容
- 不额外收集与功能无关的个人信息

## 1. 我们会处理哪些信息

为了让本应用正常运行，我们可能会在你的设备本地处理和存储以下信息：

- 会话标题、聊天记录、消息内容
- 你填写的 API Endpoint、模型名称等接口配置
- 语言偏好、首次风险确认状态及其他应用设置
- 你填写的 API Key

其中：

- 聊天记录、会话信息和一般设置会保存在本地应用数据中
- API Key 会保存在 iOS Keychain 中，而不是以明文形式保存在普通文件中

## 2. 我们不会主动做什么

在当前版本中：

- 我们不会为你创建账号体系
- 我们不会将你的聊天记录上传至开发者自建服务器
- 我们不会将你的聊天内容默认同步到云端
- 我们不会在 `SKILL.md` 原文之外，额外插入隐藏路由、隐藏预处理或秘密提示内容后再发送给模型

## 3. 当你发送消息时，会发生什么

当你在本应用中发起一次对话请求时，本应用会将以下内容直接发送到**你自行配置的第三方模型服务接口**：

- 应用内置的 `SKILL.md` 原文
- 当前会话中的历史消息
- 你填写的模型名称、接口地址及 API Key

这些请求由本应用**直接发送到你配置的第三方服务商**，而不是先发送到开发者自建中转服务。

这意味着：

- 你的聊天内容是否被第三方服务商记录、分析、保留或用于模型训练，取决于该第三方服务商自己的隐私政策、数据政策与服务条款
- 你需要自行判断所配置服务商的合规性、稳定性和安全性

## 4. 导出与分享

本应用支持将聊天内容导出为 JSON、Markdown 或 JPG。

当你使用导出功能时：

- 文件会先在本地生成
- 分享通过 iOS 系统分享面板完成
- 是否发送给第三方、发送给谁、发送到哪里，由你自行决定

我们不会因为你点击导出，而自动将导出内容上传到开发者服务器。

## 5. 网络权限与连接检测

为了提升连接体验，本应用可能会在首次使用、保存配置或正式发起请求前，对你填写的接口地址进行轻量级连接准备或可达性检测。

这类行为仅可能用于：

- 触发本地网络权限弹窗（如适用）
- 预热网络连接
- 判断接口是否基本可访问

这类请求不以收集你的聊天内容为目的。

## 6. 数据删除与控制

你可以在应用内自行管理本地数据，包括但不限于：

- 清空当前会话
- 删除历史会话
- 恢复或彻底移除已删除内容

由于当前版本主要采用本地存储模式，你在设备上删除相关内容后，相关数据通常也会从本地应用数据中被移除。

## 7. 未成年人说明

如果你是未成年人，请在监护人指导下使用本应用，并在使用任何第三方 API 服务前充分了解其费用、条款与风险。

用户需自行承担因使用第三方 API 服务而产生的：
